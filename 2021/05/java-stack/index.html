자료구조 - Stack

2021-05-07
⏱ 10 min.
datastructure

Stack은 사전적으로는 ‘더미’를 의미한다. 이 개념을 처음 공부할때 데이터를 수직으로 관리하는 자료구조라고 생각하며 이해했다.

Stack은 먼저 들어온 데이터가 나중에 나간다고 해서 FILO(First In Last Out)의 형태를 띈다고 한다.

Stack의 특징은 다음과 같다.

  • 먼저 들어간 데이터가 나중에 나간다. First In Last Out
  • 인터럽트 처리, 수식의 계산, 서브루틴의 복귀 번지 저장등에 쓰인다.
  • 그래프의 깊이 우선 탐색(DFS) 알고리즘에 사용됨
  • 재귀적(Recursion) 함수를 호출할 때 사용

아직은 Stack의 특징이 와닿지 않는다. 앞으로 자료구조 공부를 하면서 계속 채워넣어야겠다.

실제 Java로 Stack을 사용하는 예제 코드를 보자.

Stack은 java.util에 있는 클래스이며, Vector 클래스를 구현하는 제네릭 클래스이다.

Vector 클래스가 무엇인지는 아직 잘 모르겠지만, List<E>를 구현하는 클래스임을 알 수 있다.

데이터 삽입

Stack에서 데이터를 삽입하는 방식은 add(), push()가 있다. add()와 push()는 어떤 차이가 있을까?

add()는 Vector 클래스의 메소드인데, 이 메소드는 Collections 객체의 add()를 오버라이딩 한듯하다.

push()는 Stack 클래스에 선언된 메소드이다. 자료구조에 데이터를 삽입(addElement())하고, 삽입한 데이터를 반환하는 메소드이다.

로직이 다를뿐 결국 데이터를 삽입하는 측면에선 똑같이 동작하므로 예제코드에서는 Stack 클래스의 기능을 최대한 활용하겠다.

데이터를 삽입하고, 콘솔에 출력해보았다. 정상적으로 확인이 되었다.

데이터 삭제

데이터 삭제를 돕는 함수는 remove()와 pop()이 있다.

remove()는 Vector 클래스의 메소드이다. index를 파라미터로 넘겨받아서 해당 인덱스의 데이터를 삭제하는 일을 수행한다.

pop()은 Stack 클래스의 메소드이다. peek() 이 언급되서 확인해보니

peek() 역시 Stack 클래스의 메소드로서, 자료구조의 최상단 데이터를 반환하는 메소드이다.

다시 pop()으로 돌아가면, pop()은 자료구조의 마지막 인덱스(len-1)의 데이터를 삭제(removeElementAt())하고, 삭제된 마지막 데이터(peek())를 반환하는 메소드라고 할 수 있다.

pop()이 실행되면서 삭제할 데이터를 반환받고, 실제 자료구조에서 삭제된것까지 확인했다.

정리

데이터 삽입

  • Stack은 데이터를 삽입할 때 push() 를 이용한다.

  • push()는 데이터를 삽입하고, 이 때 삽입할 데이터를 반환한다.

  • add()는 boolean 타입의 메소드이므로 데이터 삽입가능여부를 확인할때 이용할 수 있을것 같다.

데이터 삭제

  • Stack에서 데이터를 삭제할땐 pop()을 이용한다.
  • pop()은 데이터를 삭제하고, 삭제하는 데이터를 반환한다.
  • remove()는 특정 인덱스의 데이터를 삭제할때 이용한다.

기타 메소드

  • peek()
    • 가장 최상단의 데이터를 반환하는 메소드
  • empty()
    • Stack이 비어있는지 여부(boolean) 를 반환하는 메소드
  • size()
    • Stack의 길이(int)를 반환하는 메소드
    • Vector 클래스의 메소드
  • contains(data)
    • 파라미터로 들어온 데이터가 Stack에 있는지 여부(boolean)을 반환하는 메소드
    • Vector 클래스의 메소드
